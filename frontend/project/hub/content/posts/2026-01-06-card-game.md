## 一. 最初: 关于卡牌游戏的最初设想

第一次接触卡牌游戏, 还是从昆特牌开始, 昆特牌最打动我的, 不是数值, 而是两个人之间的心里博弈. 或者是针对对方的阵容做出有针对的部署. 一次次的挑战到最终成功. 针对十张手牌的合理部署是关键.有一张掌控全局的感觉.

后续在读明朝历史时, 就希望能为这段历史做一个卡牌游戏. 最喜欢的卡牌和最喜欢的历史结合还有什么比这更酷的.

“洪武”是我为卡牌起的名称, 以三种操作为原则, 我为这个卡牌设计了大量复杂的规则:

1. 牌的复杂:有帝王牌(各皇帝),指挥牌(文臣武将),技能牌(战术天气)等
2. 兵种的复杂: 骑兵,重甲兵,轻步兵,弓箭手,火铳手,战车等
3. 战斗过程的复杂: 通过打出牌而影响士兵, 士兵通过不同的操作(进攻/防守/休养)进行对冲结算
4. 复杂的结算规则: 各个兵种的派出数量, 当前的战斗值, 都会影响对冲结果的计算.
5. 对局的复杂: 局数, 每局的时间, 还有每局的伤害变化从而对局的节奏变化.

当规则越来越多, 我发现我无法知道这些规则是否有想象中的那么好玩, 更没有办法在这些规则中做出平衡. 每一条规则的增加都意味着指数型增加的复杂性.

## 二. 开场: 从一个可玩的原型开始

考虑到自己的方案过于复杂, 在和ChatGPT一遍遍讨论后得出结论, 必须按照最小可玩原则完成一个demo, 然后发布到线上进行反馈.

现在就有了一个最小可玩原则的demo. 这个原则只保留了一件事:

**_同时出招 + 猜对方_**

我刻意去掉了所有复杂的地方, 兵种, 技能, 筹拍, 卡组构成, 我只想验证三种操作是否有“骗取”, “爆发”和“节奏”

1. 整体流程为: “选择房间/创建房间 - 对战 - 小局计算 - 整体结算 - 再来一局”
2. 回合操作规则: 进攻 / 防守 / 休养（attack/defend/rest）进行计算
3. 回合计算规则:

| P1 / P2 的Hp变化 | attack | defend | rest   |
| ---------------- | ------ | ------ | ------ |
| attack           | -2, -2 | -1, 0  | +1, -2 |
| defend           | 0, -1  | 0, 0   | 0, +1  |
| rest             | -2, +1 | +1, 0  | 0, 0   |

1. 对战结束规则: 最多 10 回合，或任一方 HP 归零

这就是当前的最小可玩原则. 最核心的是回合计算规则. 三种操作直接的互相结算. 如何保持平衡与可玩性.

## 三.过程: 规划规则遇到的问题

当前是真的不好玩, 没啥有意思的game点.就是简单的石头剪刀布. 并在回合计算规则上花费了好久, 遇到了四种情况:

1. **矩阵不对称:** 我一开始靠感觉调数值, 后来发现 A 打 B 和 B 打 A 竟然不是镜像, 说明规则本身就不稳定. 于是我把所有组合写成矩阵, 强迫自己检查对称性和总收益.
2. **rest 过强:** 能回血意味着“拖”是正收益, 玩家就会倾向保守. 结果就是对局不推进, 进攻没有存在感.
3. **defend 过强:** 我砍掉回血后, 防守变成最稳策略, 因为它风险最低, 收益不差. 对局又变成“谁先动谁吃亏”.
4. **attack 过强:** 我把进攻收益拉高, 立刻出现另一种极端: 谁都想抢先手, 策略空间反而更小.

经过遇到的这些问题, 认识到卡牌游戏平衡性的重要. 后续后进行代码复查, 数学层面的游戏性平衡优化并一直和大家分享.

## 四.预告: 后面的规划

后面的规划分成几个方面:

1. 关于卡牌的核心玩法需要优化, 当前的单张牌, 变为5张牌, 可选三张牌进行对决.每张牌互相对决.
2. 加入技能牌: 技能牌/或者将令牌, 不加入对决, 影响当前对决过程.
3. 可以改为每张牌的武力值, 或者是进攻值. 不再有防守值, 每张牌但单纯进攻对冲结算.
4. 没想好, 还是没想好怎么规划后面的打牌流程, 需要再次和ChatGPT梳理.
