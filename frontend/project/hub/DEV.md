# DEV.md

## 目标
根据设计截图整理页面与交互需求，作为实现与验收的开发说明。

## 站点结构与导航
- 顶部居中导航：`首页 / 博客 / 产品`
- 站点风格：极简、留白充足、单栏内容区居中。
- 所有页面底部（或内容区尾部）提供外链：`rss` 与 `github`，前置 “↗” 图标。

## 页面与内容

### 1) 首页（个人介绍）
- 标题：`个人介绍`
- 一段自我介绍文本（多行段落）。
- 文章列表（示例两条）：
  - `2026年1月6日` — `想做一个卡牌游戏(一)`
  - `2024年1月9日` — `Embracing Vim: The Unsung Hero of Code Editors`
- 文章条目布局：日期在左，标题在右，同一行对齐。
- 产品列表（与产品页一致的卡片样式，展示若干条）。

### 2) 博客列表页
- 标题：`博客`
- 文章列表与首页一致的样式与排版（日期左、标题右）。
- 文章不进行分类，博客页显示全部文章。

### 3) 产品列表页
- 标题：`产品列表`
- 产品卡片列表（示例仅一张）：
  - 卡片包含：封面图、产品名称、简短描述。
  - 示例标题：`卡牌游戏在线 Demo`
  - 示例描述：`即时对战的卡牌策略玩法，支持在线体验与对战...`
  - 点击卡片进入产品详情页。

## 视觉与排版
- 全站以白色背景为主，文本使用深灰/黑。
- 版心偏窄，内容左对齐但整体居中（视觉上中轴对齐）。
- 标题字号明显大于正文。
- 列表间距适中，页面留白明显。
- 使用 Tailwind CSS 进行样式实现，参考提供的版式风格（极简、紧凑排版、细字体重过渡）。

## 交互细节
- 文章标题、导航项均为可点击链接。
- RSS / GitHub 为外链，前置 “↗” 图标。

## 内容来源与构建
- Markdown 文章存放目录：`content/posts/`。
- 文章元信息建议使用 frontmatter（至少包含：`title`、`date`、`slug`、`summary`）。
- 博客列表在构建时扫描 md 并生成（或构建时直接读取），按日期倒序排列。
- 若文章仅来自仓库内 md 文件，默认不需要额外构建服务；更新文章需重新构建/发布。
- 若希望后台在线新增/修改文章且无需重新部署，则需要引入构建服务或后端 CMS（待确认）。
- Vite 配置：不引入额外插件与配置，采用最简单方式读取 md（建议 `import.meta.glob` + `?raw`）。
- Markdown 渲染采用方案 3：引入 `marked` 作为运行时解析依赖，保证语法完整，无需 Vite 插件与额外配置（仅写入 `package.json`，安装由使用者自行执行）。
- 产品数据来自前端数组（非 md），包含：标题、介绍、地址等字段。
- Tailwind CSS：仅写入依赖与基础配置文件，不自动安装。
  - 依赖（已写入 `frontend/package.json`）：`tailwindcss`、`postcss`、`autoprefixer`。
  - 配置文件：`tailwind.config.cjs`、`postcss.config.cjs`（位于项目目录）。

## 路由与页面形态（单页面）
- 站点为单页面应用（SPA），所有路由由前端接管。
- 路由路径基于站点前缀：`/hub`。

### 路由清单
- `/hub/`：首页，包含「文档列表」与「产品列表」两个列表。
- `/hub/blogs`：文档列表页。
- `/hub/blogs/[blog-id]`：文章详情页。
- `/hub/products`：产品列表页。
- `/hub/products/[product-id]`：产品详情页，包含产品介绍、产品地址等信息。
- 其余未匹配路由进入 404 页面。

## 404 页面
- 当访问未定义路由时展示 404 页面。
- 页面包含返回首页按钮，跳转到 `/hub/`。
